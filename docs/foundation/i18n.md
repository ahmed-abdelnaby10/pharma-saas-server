# I18n Foundation

## Purpose

Provide a simple, production-sane translation foundation for backend responses using `Accept-Language`, request-scoped translation helpers, and shared `en` / `ar` locale files.

## Dependencies

- `src/core/config/app.config.ts`
- `src/core/i18n/locales/en/*`
- `src/core/i18n/locales/ar/*`
- `src/shared/middlewares/language.middleware.ts`
- `src/shared/middlewares/error.middleware.ts`

## Endpoints

No new endpoints in this slice. Existing foundation responses are now localized.

## Headers

- `Accept-Language`: optional. The resolver checks request language entries in order and accepts `en`, `ar`, and regional variants like `en-US` or `ar-EG`.
- `x-request-id`: unchanged.

## Path Params

None.

## Query Params

None.

## Request Body

None.

## Response Shape

Success responses keep the same normalized shape:

```json
{
  "success": true,
  "message": "OK",
  "data": {}
}
```

Error responses keep the same normalized shape:

```json
{
  "success": false,
  "message": "Route not found",
  "errorCode": "ROUTE_NOT_FOUND",
  "requestId": "..."
}
```

Only the `message` value changes by locale.

## Validation Rules

- Missing translation keys fall back to English.
- Missing English keys fall back to the key string itself.

## Future Module Translation Keys

- Add new keys inside the matching namespace file under both `src/core/i18n/locales/en/` and `src/core/i18n/locales/ar/`.
- Keep keys namespaced by module, for example `inventory.batch_created` or `pos.sale_finalized`.
- Controllers and services should prefer passing translation keys through shared errors or `req.t(...)` instead of hardcoding user-facing text.

## Permissions Required

None.

## Tenant / Branch Scope Rules

- This slice does not resolve tenant default language yet.
- Current fallback order is:
  1. first supported language from `Accept-Language`
  2. app default language

## Side Effects

- `req.lang` is populated for every request.
- `req.t(key, params)` and `req.translate(key, params)` are available for controllers, services, and middlewares.
- Shared errors can carry translation keys and are translated centrally in the error middleware.
